# Distributed Systems Basics

Multiple machines working together as a single system.

---

## Why?

- Scalability
- Fault tolerance
- High availability

---

## Key Characteristics

- Multiple nodes communicating over a network
- Data distributed across nodes
- No single point of control

---

## Challenges

- Network latency
- Partial failures — some nodes die, others don't
- Data consistency across nodes
- Coordination between nodes gets complex fast

---

## Real-World Usage

Netflix, Uber — both run on distributed systems at massive scale.

---

**Distributed systems = scale and resilience. Trade-off is complexity in coordination and consistency.**
